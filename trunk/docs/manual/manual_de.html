I. Vorwort

Als Programmierer hat man es immer schön einfach, man weiß was man gemacht hat und wie man es nutzen kann bzw. benutzen muss, das es richtig funktioniert. Doch was ist nun wenn jemand keine Ahnung hat wie das ganze aufgebaut ist? Der hat ein Problem und ist auf sich allein gestellt! So etwas muss aber nicht sein, denn alles lässt sich hoffentlich erklären. Aus diesem Grund versuchen wir als freundliche Programmierer den Übergang von Unwissenheit zur Kenntnis durch dieses HOWTO so einfach wie möglich gestalten.

II. Benutzerfunktionen

III. 1. Anmelden

Um sich in das Admin-Interface einzuloggen, öffnet man die Unterseite /admin(z.B.: www.server.de/ => www.server.de/admin/ oder www.server.de/comacms/ => www.server.de/comacms/admin/). Dort gibt man seinen Loginnamen(den Nick, nicht den Anzeigenamen!) und sein Passwort ein und klickt anschließend auf Login. Wenn alle eingaben korrekt waren ist man nun im Admin-Interface.

III. 2. Abmelden

Einfach den Menüpunkt Logout (ganz unten im Admin-Interface) anklicken.

Nach vier Stunden wird man automatisch ausgeloggt.

IV. Grundfunktionen verwenden

IV. 1. Texte

IV. 1. 1. Texte erstellen

Den Menüpunkt Seiten-Struktur auswählen, dann auf den Button mit der Aufschrift "Neue Seite erstellen" klicken. Mit dem nun erschienen Dialog wird die neue Seite angelegt.

Durch das Klicken auf den Button mit der Aufschfift "Erstellen" wird die Seite erstellt.

Erläuterung zu den Eingaben

Name/Kürzel
Der Name/das Kürzel dient zum logischen und eindeutigen identifizieren der Seite. Mit ihm greift man auf die Seite zu.
Titel
Der Titel wird für gewöhnlich in der Titelleiste des Browsers angezeigt und hat eine Art Überschriftfunktion.
Seiten-Typ
Über den Seiten-Typ bestimmt man was für eine Seite erstellt wird. Um einen Text zu erstellen muss man natürlich hier die Eigenschaft "Text" auswählen.
Sprache
Die Sprache beschreibt in welcher Sprache der Text verfasst wird.
Zugang
Über die Eigenschaft Zugang kann man bestimmen, wer später den Text lesen können soll.
  • Öffentlich: Jeder kann den Text lesen
  • Privat: Nur angemeldete Benutzer können den Text lesen
  • Versteckt: Der Text ist nur eingetragen, aber noch nicht zum Lesezugriff freigegeben
Unterseite von
Mit der Eigenschaft "Unterseite von" kann man den Text in die Seitenstruktur eingliedern um die Seite ein bisschen zu strukturisieren.
Kommentar
Der Kommentar gibt an was man gemacht hat und wird nachher bei der Versionsübersicht(Übersicht über alle Veränderungen) angezeigt um leichter zu erkennen um welche Version des Textes es sich handelt.
Bearbeiten
Wenn man dieses Kästchen ausgewählt hat wird man sofort zum Bearbeitungsdialog des Textes weitergeleitet, um die Seite bearbeiten zu können.

IV. 1. 2. Texte bearbeiten

In den Seiteneditor gelangt man entweder duch die Option "Bearbeiten" beim Erstellen der Seite oder indem man in der Seiten-Struktur auf das Bearbeiten-Symbol (weißes Papier mit Stift) der jeweiligen Seite klickt.

Info: Der Seiten-Editor kann auf verschiedenen Rechnern unterschiedlich aussehen. Das liegt daran, dass der Seiteneditor im Java-Script erweitert wird, wenn es möglich ist. Wenn allerdings Javascript deaktiviert ist erscheinen grundsätzlich keine Formatierungs-Buttons.

IV. 1. 3. Texte formatieren

Um Texte zu formatieren ist in ComaCMS eine eigene Formatierungssprache implemetiert worden, mit der man Texte übersichtlicher gestalten kann, die Texte aber immer noch dem Grunddesign entsprechen.

In dieser Erklärung wird immer kurz gezeigt, wie die Formatierung aussieht und anschließend der dazugehörige Formatierungscode angezeigt.

IV.1.3.1 Schriftformatierung

Mit ComaCMS kann man fette, kursive und untersrichene Texte erstellen.
Diese Formatierungen kann man natürlich auch kombinieren.

Mit ComaCMS kann man **fette**, //kursive// und __untersrichene__ Texte erstellen.\\
Diese Formatierungen kann man natürlich auch __//**kombinieren**//__
IV.1.3.2 Links

Es gibt verschiedene Möglichkeiten Links zu integrieren, ein Teil der Links wird sogar automatisch erkannt und "klickbar" gemacht.

So werden zum Beispiel normale urls wie www.berlios.de oder developer.berlios.de in Links umgewandelt.
Man kann aber auch Links erzwingen indem man sie mit eckigen Klammern umgibt (www.berlios.de).
Mit den eckigen Klammern kann man auch einen Link umbenennen: Die Homepage von BerliOS.
Um auf interne Seiten zu verlinken nimmt man den Namen oder die ID der Seite und schreibt dann: Link auf eine interne Seite.
Auch Emailadressen können in Links umgeformt werden, sie müssen dann in spitze Klammern geschrieben werden: comacms@williblau.de

IV.1.3.5 Bilder

Wichtig ist natürlich auch die Möglichkeit zur Einbindung von Bildern in den Text.

Es gibt zwei verschiedene Möglichkeiten Bilder einzubinden.

  • mit der URL zum Bild:
img/comacms_logo.png
  • mit einer relativen Angabe zum medienordner der Homepage:
media:comacms_logo.png

Das Aussehen der Bilder kann durch verschiedene Parameter verändert werden. Es gibt zwei verschiedene Typen an Parametern, einmal die Parameter für das Aussehen und einmal die Parameter für die Größe der Bilder.

  • Aussehen
    • box: Das Bild steht in einem Kasten mit dem Titel als Bildunterschrift (Standardeinstellung)
    • box_only: Das Bild steht in einem Kasten ohne Bildunterschrift
    • picture: Das Bild steht für sich alleine

    • original: Stellt das Bild in der Originalgöße dar
    • thumb: Begrenzt die Breite des Bildes auf 180 Pixel (Standardeinstellung)
    • w80: Begrenzt die Breite des Bildes auf die angegebene Breite (in diesem Fall auf 80 Pixel)
