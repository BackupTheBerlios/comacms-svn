Comments

It is very good to comment the source code but we have to write it in one style and not sometimes so and sometimes so.

If you give a comment, which explain what the code does you should write it so: 
	//
	// This is a comment
	//
	echo 'hi im  the code, wich is explained';

If you describing a function it is good to comment the function like this 
/*****************************************************************************
 *
 *  testfunction(string text, int integer)
 *
 *  This comment discribes what the function do
 *
 *  string text		: this is the description of the parameter text
 *  int integer		: the same for the parameter integer
 *
 *****************************************************************************/
 
	function testfunction($text, $integer) {
		//
		// code
		//
	}

Strings

In PHP we have two different ways to declare strings either with single quotes or with double quotes. If your declaring a string with single quotes the compiler generates only a string. But if you declare a string with double quotes the compiler searches for variables and inserts their values into the string. Because we want a fast programm try to use single quotes so much as possible. 
	//
	// bad
	//
	$string = "A string with no variable and dosen't need to handled like one";
	
	//
	// good
	//
	$string = 'A string with no variable and is handled like one';
